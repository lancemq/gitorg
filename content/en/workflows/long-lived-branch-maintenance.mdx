# Maintaining Long-lived Branches

## Why long-lived branches get expensive

The longer a branch stays alive, the easier it is for divergence, repeated conflicts, and noisy integration history to pile up.

The real goal is not "never conflict." The goal is to keep sync batches small and regular.

## Suggested sequence

1. set a recurring sync cadence
2. keep each integration batch small
3. pay down conflict and history debt regularly

## A realistic maintenance rhythm

```bash
git switch migration/auth-refactor
git fetch origin
git rebase origin/main
git status
git log --oneline --decorate -8
```

After each sync, validate the branch before drift builds again.

## Common mistakes

- waiting until the final merge to sync
- letting one long-lived branch absorb multiple unrelated themes
- syncing repeatedly but never cleaning the resulting history
