# Git References and HEAD

## Why this concept matters

Many Git commands feel confusing not because their syntax is hard, but because people mix up commits and references.

A more accurate model is:

- commit objects are history nodes
- branches, tags, and HEAD are references that point at those nodes

## What HEAD is

In normal day-to-day work, `HEAD` usually acts as a symbolic reference to the branch you are currently on.

That explains why:

- switching branches changes HEAD
- making a commit moves the current branch forward
- reset changes where the current branch and HEAD point
