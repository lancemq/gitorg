# 发布分支工作流教程

## 场景说明

梳理何时切发布分支、何时冻结功能、以及如何把修复回流主线。

## 推荐顺序

1. 从稳定主线切出 release 分支
2. 只接受发布相关修复
3. 把最终修复回合并到主线

## 为什么这个顺序更稳

工作流的价值通常不在某一条命令本身，而在于它把先看什么、再做什么、出了问题怎么回退固定下来。

## 相关命令

建议结合 git fetch、git log、git diff、git merge、git rebase 或 git cherry-pick 一起理解。
