# Fork 与上游同步教程

## 场景说明

在 fork 模式协作中维持 origin 与 upstream 的清晰边界。

## 推荐顺序

1. 把个人 fork 作为 origin
2. 把官方仓库配置为 upstream
3. 定期 fetch upstream 并同步本地主分支

## 为什么这个顺序更稳

工作流的价值通常不在某一条命令本身，而在于它把先看什么、再做什么、出了问题怎么回退固定下来。

## 相关命令

建议结合 git fetch、git log、git diff、git merge、git rebase 或 git cherry-pick 一起理解。
