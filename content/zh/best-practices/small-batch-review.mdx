# 小批量评审专题

## 核心建议

用更小的提交批次和更短的生命周期换取更快的 review 节奏。

## 为什么这件事重要

好的 Git 实践不是让命令看起来更熟练，而是让历史更可读、协作更稳定、恢复路径更明确。

## 执行清单

- 让分支生命周期尽量短
- 一次评审只承载有限范围
- 必要时拆成多条 PR

## 适合一起看的命令

通常建议同时回看 git status、git log、git diff，必要时再补上 git rebase、git merge 或 force-with-lease 相关知识。
