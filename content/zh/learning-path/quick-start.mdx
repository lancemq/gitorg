# Git 快速上手专题

## 这个专题解决什么问题

如果你刚开始学 Git，最容易陷入的困境不是命令太少，而是命令之间的顺序和边界不清楚。

这个专题把“第一次上手 Git”拆成 4 个更容易消化的步骤：

1. 准备环境并拿到仓库
2. 学会暂存和提交
3. 学会和远端同步
4. 学会在分支上开始第一轮协作

你不需要一次把所有 Git 命令学完。更稳妥的策略是先把最小闭环走通，再补更高级的话题。

## 推荐阅读顺序

### 1. [环境与 clone](/zh/docs/learning-path/setup-and-clone)

先完成身份配置、仓库获取和首次状态检查，搞清楚“仓库是怎么来到本地的”。

### 2. [暂存与 commit](/zh/docs/learning-path/stage-and-commit)

重点建立工作区、暂存区和提交历史三层模型，学会把文件改动变成清晰提交。

### 3. [远端同步](/zh/docs/learning-path/sync-with-remote)

把 `fetch`、`pull`、`push` 的职责拆开理解，先学观察，再学整合，再学发布。

### 4. [第一个特性分支](/zh/docs/learning-path/first-feature-branch)

开始进入最简单的协作节奏：创建分支、提交改动、同步主分支、准备合并。

## 这个专题最适合谁

- 第一次接触 Git 的开发者
- 会敲几条命令，但还不理解命令之间关系的人
- 经常在 `pull`、`push`、`commit` 这些基础动作上犹豫的人
- 想先建立稳定节奏，再进入 rebase、reflog、reset 等高风险主题的人

## 学这个专题时最值得记住的原则

### 先看状态，再做动作

每次关键操作前后，尽量都看一眼：

```bash
git status
git log --oneline --decorate -5
```

### 先让单人闭环稳定，再补团队协作

如果你连“改文件 -> add -> commit -> push”都还不稳，就先不要急着研究复杂冲突和历史重写。

### 先拆清动作，再记命令

比如：

- `clone` 是拿仓库
- `add` 是准备提交内容
- `commit` 是保存本地历史
- `fetch` 是同步远端信息
- `pull` 是下载并整合
- `push` 是发布本地提交

## 快速上手阶段先不要急着碰什么

在这个专题里，你可以先不急着深入：

- `rebase`
- `reset --hard`
- `cherry-pick`
- `reflog` 的恢复细节
- plumbing 命令

不是这些内容不重要，而是它们更适合建立在“基础节奏稳定”之后。

## 学完这个专题后建议继续去哪

如果这 4 篇都已经读顺了，下一步建议进入：

1. `fetch` 与 `pull` 的策略边界
2. `git rebase` 的使用边界
3. `feature branch collaboration` 工作流
