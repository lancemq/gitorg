# Git 中的引用与 HEAD

## 为什么要学这个概念

很多 Git 命令之所以难理解，不是因为参数复杂，而是因为没有把“提交对象”和“引用”区分开。

一个更准确的心智模型是：

- 提交对象本身是历史节点
- 分支、标签、HEAD 等是指向这些节点的引用

## 什么是引用

引用可以理解成“给某个提交起一个更容易记住的名字”。

比如：

- `main`
- `feature/login`
- `v1.0.0`

这些名字背后，最终都指向某个提交。

## HEAD 是什么

在大多数日常场景下，`HEAD` 表示“你当前所处位置的符号引用”。

通常它会指向当前分支，而当前分支再指向某个提交。

这也是为什么：

- checkout 不同分支时，HEAD 会变化
- commit 之后，当前分支会向前移动
- reset 会让当前分支和 HEAD 指到别的位置
