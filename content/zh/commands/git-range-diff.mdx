# git-range-diff 教程

## 一句话理解

git-range-diff 用于比较两组提交序列之间的差异。

## 什么时候适合用

- 当你需要比较两组提交序列之间的差异
- 当你想把这类操作做成稳定流程而不是手工重复
- 当你需要更准确地理解 Git 在这一步到底记录了什么

## 基本示例

```bash
git range-diff origin/main...v1 origin/main...v2
```

## 读这条命令时最该注意什么

高级命令不是日常主流程，但一旦用错，恢复成本会明显升高。

## 一个更稳的实践建议

在真实仓库执行前，先用一段最小可复现历史做演练。

## 补充理解角度

- 处理复杂协作和历史分析问题
- 把一次性操作收敛成可重复流程
- 降低高风险操作的偶然性

## 延伸阅读

继续搭配 git status、git log、git show 一起看，通常更容易判断这条命令对历史、索引和工作区分别造成了什么影响。
