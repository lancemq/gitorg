# Git Org Academy 设计与交互说明

## 1. 设计目标

网站的设计目标不是传统博客，也不是工具官网，而是“内容型学习站”。

整体应同时具备：

- 文档站的清晰结构
- 学院型产品的引导感
- 工具型知识库的检索效率

## 2. 页面类型

### 2.1 首页

首页是导览页，不是内容堆叠页。

首页应承担：

- 品牌表达
- 核心价值传达
- 学习起点引导
- 频道入口分发

首页不应承担：

- 全量文档列表
- 过长的二级目录
- 大段开发说明或实现说明

### 2.2 频道页

频道页需要承担：

- 频道定位说明
- 推荐学习顺序
- 主题卡片聚合

频道页应该比首页更完整，但仍然不是详情页。

### 2.3 详情页

详情页负责：

- 完整阅读
- 信息深挖
- 相关推荐
- 顺序学习

## 3. 视觉原则

### 3.1 标题系统

- 标题应简洁有力
- 当前全站标题尺寸已压缩，避免过强营销感
- `h1/h2` 更接近文档站而不是落地页

### 3.2 信息密度

- 首页信息密度低于频道页
- 频道页信息密度低于详情页
- 侧边菜单优先稳定，不平铺全部二级内容

### 3.3 卡片与面板

- 首页以导览卡片和模块面板为主
- 频道页以主题卡片和学习顺序模块为主
- 详情页以正文面板和支持模块为主

## 4. 双语设计原则

### 4.1 路由独立

- 中文和英文各有独立路由
- 不采用页面内即时翻译切换模式

### 4.2 语言切换器

语言切换器应遵循：

- 中文按钮始终显示 `中文`
- 英文按钮始终显示 `English`
- 标签和按钮都带有语言标识图标

### 4.3 文案一致性

- 中英文信息架构必须一致
- 中文和英文不要求逐字翻译，但必须表达同一信息层次

## 5. 搜索交互原则

搜索应是“快速入口”，不是另一个目录页。

因此当前采用：

- 弹层模式
- 快捷键打开
- 分类筛选
- 结果分组
- 最近访问
- 常用入口

搜索结果卡片必须：

- 标题清晰
- 摘要简短
- 标注频道来源

## 6. 正文展示原则

### 6.1 MDX 正文

- 正文只保留页面头部主标题
- MDX 内部的 `h1` 不重复渲染
- 正文章节标题允许正常阅读，不应被过度截断

### 6.2 图例与解释

对以下内容，应优先考虑图例辅助：

- 历史变换类命令
- 多层状态类命令
- 协作流程类命令

当前已适合配图的命令包括：

- rebase
- merge
- reset
- stash

## 7. 导航与滚动原则

- 左侧菜单必须独立滚动
- 一级入口尽量稳定
- 不在侧边栏直接平铺过长命令列表
- 当前页只高亮频道级入口

## 8. 品牌表达

当前首页 slogan：

- 中文：`学会 Git，写出更好的历史。`
- 英文：`Master Git. Build Better History.`

这一组品牌语气应保持：

- 简洁
- 有方向感
- 与网站的“命令 + 工作流 + 原理”结构一致
