# GitOrg Atlas 后续路线图

## 1. 当前阶段判断

网站已经从单页说明型项目发展为一套较完整的双语内容站。

目前重点已经不是“是否能用”，而是：

- 如何让内容继续高质量扩容
- 如何保持导航和信息架构稳定
- 如何提升搜索、推荐和学习路径体验

## 2. 近期优先级

### 2.1 内容深化

优先继续强化高价值命令专题：

- rebase
- reset
- stash
- cherry-pick
- worktree
- submodule

建议继续加入：

- 图例
- 对比表
- 风险边界
- 恢复路径

### 2.2 学习路径强化

建议把 Learning Path 再做得更清晰：

- 给每篇学习路径内容加阶段编号
- 增加“下一步学什么”
- 让频道页显示更明确的推荐顺序

### 2.3 搜索增强

后续可以继续补：

- 结果权重优化
- 无结果推荐词
- 热门搜索
- 更明确的分类排序

## 3. 中期方向

### 3.1 内容系统自动化

当前内容已经很多，后续建议推进：

- 自动内容索引
- 自动统计各频道数量
- 自动生成相关推荐关系
- 自动生成 sitemap 级内容映射

### 3.2 SEO 与分发

建议进一步增强：

- 更细粒度 metadata
- 结构化数据
- 专题页 SEO 文案
- 内容页 canonical 与关联关系

### 3.3 视觉一致性

建议持续收敛：

- 频道页头图结构
- 图例组件体系
- 卡片样式层级
- 代码块、表格、提示块样式

## 4. 长期方向

### 4.1 成为系统化 Git 学习站

长期目标不是单纯“教程很多”，而是：

- 有路径
- 有层级
- 有原理
- 有协作语境

### 4.2 建立更强内容运营能力

后续可以考虑：

- 专题合集页
- 命令对比页
- 新手专题路线
- 团队实践路线
- 风险操作合集

### 4.3 建立更强的可视化解释能力

Git 很多难点来自抽象结构，因此后续最值得投资的是：

- 历史图
- 对象图
- refs / HEAD 关系图
- reset / rebase / merge 对比图

## 5. 演进原则

后续所有新增工作，建议优先满足这三条：

- 新内容要能放进现有频道体系
- 新页面不能破坏导航一致性
- 新功能要优先强化学习和检索效率，而不是增加噪音
